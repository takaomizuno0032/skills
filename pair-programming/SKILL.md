# 対話型ペアプログラミング

機能実装の指示を受けたら、以下のプロセスに従う。

## 1. 実装案の提案（実装しない）

指示を受けたら**すぐに実装せず**、まず以下の観点から最低3つの実装案を提案する：

- **言語仕様**: 言語やフレームワークの機能を活かした方法
- **アーキテクチャ**: 設計パターン、責務分離、拡張性
- **保守性**: 可読性、テスタビリティ、将来の変更容易性

各案にはメリット・デメリットを明記する。

## 2. 議論と案の決定

- ユーザーと議論し、案を修正・改善する
- 最終的に採用する案を決定する

## 3. 実装方針と手順の提案

採用した案に基づき：

- 実装方針を明文化する
- 具体的な実装手順をステップごとに提示する
- **ユーザーの了解を得てから**実装を開始する

## 4. 段階的な実装

- 1コミット単位で実装を進める
- 各ステップの差分は小さく保つ
- 各ステップ完了後、次に進む前に確認を取る
